🎯 4 PROVIDERS YANG SIAP DIGUNAKAN:

1️⃣  AuthProvider
    Purpose: Mengelola autentikasi dan data user
    
    Properties:
    ├─ user: User? (data user yang login)
    ├─ isAuthenticated: bool (status login)
    ├─ isLoading: bool (status loading API)
    └─ errorMessage: String? (pesan error)
    
    Methods:
    ├─ login(email, password) → Future<bool>
    ├─ signup(name, email, password) → Future<bool>
    ├─ logout() → Future<void>
    └─ clearError() → void
    
    Usage:
    ```dart
    context.read<AuthProvider>().login(email, password);
    context.watch<AuthProvider>().isAuthenticated;
    ```

2️⃣  NavigationProvider
    Purpose: Mengelola state bottom navigation
    
    Properties:
    ├─ currentIndex: int (halaman yang aktif)
    └─ pageController: PageController (animasi halaman)
    
    Methods:
    ├─ onTapNav(index) → void
    └─ onPageChanged(index) → void
    
    Usage:
    ```dart
    Consumer<NavigationProvider>(
      builder: (ctx, nav, _) {
        return CustomBottomNav(
          currentIndex: nav.currentIndex,
          onTap: nav.onTapNav,
        );
      },
    )
    ```

3️⃣  ProgramProvider
    Purpose: Mengelola data program lari/fitness
    
    Properties:
    ├─ programs: List<RunningProgram> (daftar program)
    ├─ selectedProgram: RunningProgram? (program terpilih)
    └─ isLoading: bool (status loading)
    
    Methods:
    ├─ fetchPrograms() → Future<void>
    ├─ selectProgram(program) → void
    ├─ completeProgram(id) → void
    ├─ addProgram(program) → void
    └─ deleteProgram(id) → void
    
    Usage:
    ```dart
    context.read<ProgramProvider>().fetchPrograms();
    context.watch<ProgramProvider>().programs;
    ```

4️⃣  AppProvider
    Purpose: Mengelola settings aplikasi umum
    
    Properties:
    ├─ isDarkMode: bool
    └─ appLanguage: String (id/en)
    
    Methods:
    ├─ toggleDarkMode() → void
    ├─ setDarkMode(bool) → void
    └─ setLanguage(String) → void
    
    Usage:
    ```dart
    context.read<AppProvider>().toggleDarkMode();
    context.watch<AppProvider>().isDarkMode;
    ```

🔧 CARA MENGGUNAKAN:

Pattern 1: Aksi Sederhana (tidak perlu rebuild)
    ElevatedButton(
      onPressed: () {
        context.read<AuthProvider>().logout();
      },
      child: Text('Logout'),
    )

Pattern 2: Display Data (dengan rebuild otomatis)
    Consumer<AuthProvider>(
      builder: (context, auth, _) {
        return Text('${auth.user?.name}');
      },
    )

Pattern 3: Listen Property Tertentu (optimal performance)
    Selector<ProgramProvider, int>(
      selector: (_, p) => p.programs.length,
      builder: (_, count, __) => Text('$count programs'),
    )

Pattern 4: Multiple Providers
    Consumer2<AuthProvider, ProgramProvider>(
      builder: (context, auth, program, _) {
        return Column(
          children: [
            Text('User: ${auth.user?.name}'),
            Text('Programs: ${program.programs.length}'),
          ],
        );
      },
    )

💡 TIPS & BEST PRACTICES:

1. Testing Providers
   ├─ Test providers terpisah dari UI
   ├─ Use mockito untuk mock dependencies
   └─ Write unit tests untuk business logic

2. Error Handling
   ├─ Always wrap API calls dalam try-catch
   ├─ Set errorMessage untuk ditampilkan ke user
   ├─ Clear error setelah ditampilkan
   └─ Implement retry logic

3. Performance
   ├─ Use Selector untuk listen specific properties
   ├─ Avoid rebuilding unnecessary widgets
   ├─ Cache data dengan local storage
   └─ Paginate large lists

4. Debugging
   ├─ Print state changes di notifyListeners()
   ├─ Use Provider DevTools extension
   ├─ Check widget rebuild with paint boundaries
   └─ Monitor memory usage

🆘 HELP & SUPPORT:

Error: "Target of URI doesn't exist"
→ Solution: Run `flutter pub get`

Widget tidak rebuild saat provider berubah
→ Solution: Use Consumer<Provider> atau context.watch<Provider>()

Too many rebuilds / performance issues
→ Solution: Use Selector<Provider, SpecificType>() untuk listen specific data

Perlu akses provider di luar Consumer
→ Solution: Use context.read<Provider>() dalam methods/callbacks

API integration problems
→ Solution: Check STATE_MANAGEMENT_GUIDE.md bagian API Integration
